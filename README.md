# 불편한 시간만 알려주세요

회의 시간 조율 · **참석자 화면** — 토스 프로덕트 디자이너 챌린지 2026

**▶ Live demo:** https://besoluble.github.io/toss-pd-challenge/

반응형(모바일·데스크톱) · 실제 인터랙션 동작 · 필수/선택 참석자 두 저니 모두 시연

---

## 문제

6명이 회의를 못 잡는 진짜 이유는 *빈 시간이 없어서*가 아니다. 각자의 사정(점심 직후 회피, 특정 요일 불가…)이 **캘린더에 없고 주관적**이라, 주최자가 **상황을 이해할 수단이 없다.** 그래서 '괜찮은' 절충에 도달하지 못한다.

특히 "웬만하면 점심 직후는 피하고 싶다" 같은 **약한 선호**는 말로 꺼내기 곤란해(까다로워 보일까 봐) 아예 삼켜지고, 그만큼 주최자의 안개는 짙어진다.

## 해결

- **캘린더의 확정 일정은 시스템이 자동으로 제외** → 참석자는 *캘린더에 없는, 나만 아는 사정*만 표현
- '가능/불가능' 이진이 아니라 **'괜찮음의 정도'** 로: `편해요 · 살짝 피하고 싶어요 · 웬만하면 피하고 싶어요 · 안 돼요`
- 말이 아니라 **슬라이더로 흘리듯**, **사적으로(아무에게도 안 보임)** → 말하기 곤란한 약한 선호도 정직하게 표출
- **요일도 같은 축**(탭하면 그 요일의 슬라이더) · 필수/선택은 주최자가 정한 가중치로 처리

## 왜 이렇게 설계했나

- 완전 자동화(답만 툭 제시)는 신뢰되지 않는다 → **상황을 먼저 이해시켜야 답이 믿긴다**
- 슬라이더 **최대치 = 거절(하드)**, 그 외는 소프트 선호 → 모두가 최대로 밀어도 가능한 시간이 사라지지 않게
- 참석자의 부담은 최소로(자기 것만, 5초), 주최자의 이해는 최대로

## 기술

정적 단일 파일(`index.html`) · 의존성 없음 · Pretendard · GitHub Pages 호스팅.
인터랙션 폴리시는 [`make-interfaces-feel-better`](https://skills.sh/jakubkrehel/make-interfaces-feel-better) 원칙 적용(동심원 radius, 그림자>보더, scale-on-press, 스태거 등장, 40×40 히트영역 등).
